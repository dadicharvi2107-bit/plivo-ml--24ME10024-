# NOTES

Our best configuration scales the baseline model to 6 layers, 8 heads, and a sequence length of 256, while staying strictly under the 2,000,000 parameter limit at 1,937,920 parameters. We enable weight tying (sharing weights between the token embeddings and the output head projection) to save 40,960 parameters, which allows us to invest in a deeper model architecture. The sequence length is doubled to 256, allowing the model to look twice as far back into context, which is highly beneficial for Hindi text where characters consume 3 bytes each. We optimize the training schedule using cosine learning rate decay with a linear warmup phase of 200 steps and a tuned peak learning rate of 6e-4. The warmup phase stabilizes gradient updates at the start of training, and the cosine decay helps the model converge into a flatter, more generalizable local minimum. We implement weight decay splitting, applying a weight decay of 0.1 only to 2D weight matrices while keeping it at 0.0 for biases, norms, and embeddings to prevent underfitting. We double the batch size to 16, which increases gradient stability and exposes the model to more tokens per optimizer step within the 2,000-step budget. Finally, we optimize CPU execution by limiting PyTorch to 8 threads, which reduces thread contention and speeds up training by more than 2x.
