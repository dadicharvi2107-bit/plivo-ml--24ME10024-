Optimized Training Setup
  • Weight Tying: Enabled (tok_emb and head projection share weights, saving 40.9K parameters)
  • Learning Rate Schedule: Cosine decay with 200-step linear warmup
  • Peak Learning Rate: 6e-4 (optimized for cosine schedule)
  • Weight Decay Splitting: 0.1 weight decay on 2D weights, 0.0 on 1D biases/LayerNorms/embeddings
  • Batch Size: 16 (doubled from baseline's 8)
  • CPU Threading: Hard-limited to 8 threads (2x faster than default 16 threads due to lower contention)

* Critical Engineering Insights:
Run 2: Weight tying reduces active parameter count. When model size is held constant, restricting capacity slightly degrades performance. However, this is a necessary pre-requisite for model scaling.
Run 3: Introducing cosine decay with warmup reduces the average learning rate by more than half. For tiny models (1.3M), a flat 3e-4 is barely sufficient; decaying it leads to severe underfitting. We corrected this in Run 4 by tuning the peak LR upward to 1e-3, leading to our biggest performance breakthrough (2.2391 bpb).

Engineering Analysis

Why this Architecture is the Best Choice
  • Weight Tying & Scaling: Tying the embeddings and final head removes redundant parameters, allowing us to increase the depth of the model from 4 to 6 layers and attention heads from 4 to 8. This deeper model learns more hierarchical syntactic and semantic patterns.
  • Longer Context (256 block size): The training corpus is mixed English and Hindi. Hindi characters are encoded using UTF-8, which takes 3 bytes (tokens) per character. A block size of 128 only spans 42 Hindi characters. Extending the block size to 256 quadruples the byte span, allowing the attention mechanism to capture long-range linguistic patterns in both languages.
  • Learning Rate Warmup & Cosine Schedule: Standard GPTs require a stable training schedule. Warmup prevents early gradient explosion in the deeper network, while the cosine schedule decays the learning rate to settle cleanly in a flat local minimum for better generalizability.
  • Weight Decay Splitting: Biases and LayerNorm weights have very few parameters and are responsible for matching activations. regularizing them pulls them to zero unnecessarily. Isolating weight decay only to 2D projections prevents model underfitting.
  • CPU Thread Limit Optimization: Benchmarking revealed that 8 threads is 2.2x faster than 16 threads. This CPU profiling discovery allowed us to run larger configurations with more data (batch size 16) within our training time window.
